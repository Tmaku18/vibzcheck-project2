# Vibzcheck

Collaborative music session app built with **Flutter + Firebase** for
*Mobile App Development Studio* (CSC 4360 / 6370 — Project 2, CRN 13598).

> **Author:** Tanaka Makuvaza · Student ID `002252191` · M.S. Computer Science · Georgia State University

Vibzcheck lets a host start a listening session, invite friends, search and add
tracks via the Spotify Web API (proxied through Cloud Functions), vote on the
shared queue in real time, chat about the vibe, and get rule-based AI
suggestions for what to play next. A graduate-level fairness module re-ranks
the queue so quieter members still get airtime.

The full implementation roadmap lives in
[`.cursor/plans/vibzcheck_solo_plan_5a1b0201.plan.md`](.cursor/plans/vibzcheck_solo_plan_5a1b0201.plan.md)
(local-only; not committed).

## Tech stack

| Layer | Choice |
| --- | --- |
| Mobile | Flutter 3.38 / Dart 3.10 (Material 3, Android target) |
| State | `flutter_riverpod` 3.x |
| Routing | `go_router` 17.x with auth-aware redirect |
| Identity | `firebase_auth` (email + password) |
| Database | `cloud_firestore` (real-time queue, votes, chat, suggestions) |
| Files | `firebase_storage` (avatars, cached art) |
| Push | `firebase_messaging` (invites, vote rounds) |
| Server | `cloud_functions` (Spotify bridge, fairness ranking, notifications) |

All five required Firebase services are wired and exposed through Riverpod
providers in [`lib/core/firebase/firebase_providers.dart`](lib/core/firebase/firebase_providers.dart)
so any feature can grab them without touching a global singleton.

## Getting started

### 1. Prerequisites

- Flutter `>=3.38.7`
- Dart `>=3.10.7`
- Firebase CLI `>=15.x`
- FlutterFire CLI `>=1.3.x`
- Android SDK (target API 35) + a device or emulator

### 2. Clone and install

```bash
git clone https://github.com/Tmaku18/vibzcheck-project2.git
cd vibzcheck-project2
flutter pub get
```

### 3. Firebase project setup (one-time, console clicks)

The repo already includes `lib/firebase_options.dart`,
`android/app/google-services.json`, and `firebase.json` for the
**`vibzcheck-tmaku`** Firebase project. To make the app actually run end-to-end,
two services must be enabled inside that project (Firebase REST APIs require
the Blaze plan to provision these from the CLI; the console works on the free
Spark plan):

1. **Cloud Firestore** — open the
   [Firestore console](https://console.firebase.google.com/project/vibzcheck-tmaku/firestore)
   and click **Create database** → *Native mode* → region `nam5 (United States)`.
2. **Email/Password authentication** — open the
   [Authentication console](https://console.firebase.google.com/project/vibzcheck-tmaku/authentication/providers)
   and enable the **Email/Password** provider.

Then deploy the security rules and Firestore indexes from this repo:

```bash
firebase use vibzcheck-tmaku
firebase deploy --only firestore:rules,firestore:indexes,storage
```

### 4. Run on Android

```bash
flutter run                # debug
flutter build apk --release  # produces build/app/outputs/flutter-apk/app-release.apk
```

## Security notes

- `google-services.json` and `firebase_options.dart` are intentionally
  committed — Firebase API keys are not secrets; Firestore/Storage rules and
  Authentication enforce access.
- Spotify credentials never ship to the client; Cloud Functions hold them
  server-side and the app calls Functions, not Spotify directly.
